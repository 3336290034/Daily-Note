# 知识点总结

1，递归和迭代：

- 递：程序不断深入地调用自身，通常传入更小或更简化的参数，直到达到“终止条件”，在到达终止条件前，不断的[开启新的栈帧]；
- 归：触发“终止条件”后，程序从最深层的递归函数开始逐层返回，汇聚每一层的结果，达到条件，不断的[pop{fp,pc}，从栈恢复PC=LR=...]；

```Asm
00010420 <recursion>:
   10420: e92d4800  push {fp, lr}
   10424: e28db004  add fp, sp, #4
   10428: e24dd010  sub sp, sp, #16
   1042c: e50b0010  str r0, [fp, #-16]
   10430: e51b3010  ldr r3, [fp, #-16]
   10434: e3530001  cmp r3, #1
   10438: 1a000001  bne 10444 <recursion+0x24>  // 不相等则跳转到...
   1043c: e3a03001  mov r3, #1
   10440: ea000007  b 10464 <recursion+0x44>    // 只跳转，不链接，无条件跳转
   10444: e51b3010  ldr r3, [fp, #-16]
   10448: e2433001  sub r3, r3, #1
   1044c: e1a00003  mov r0, r3
   10450: ebfffff2  bl 10420 <recursion>    // 开启新的栈帧，BL表示带链接的跳转，跳前会将下一条指令的地址保存在LR寄存器
   10454: e50b0008  str r0, [fp, #-8]
   10458: e51b2010  ldr r2, [fp, #-16]
   1045c: e51b3008  ldr r3, [fp, #-8]
   10460: e0823003  add r3, r2, r3
   10464: e1a00003  mov r0, r3
   10468: e24bd004  sub sp, fp, #4
   1046c: e8bd8800  pop {fp, pc}        // 逐层弹栈返回，PC=LR=...
```
